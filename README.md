# PNU 학식 실시간 혼잡도 + 오늘의 식단

식당을 고르고 지금 혼잡도를 **1(여유)~5(혼잡)** 로 제보하면 현황 보드에 반영되고,
부산대 스마트캠퍼스 식단안내와 연동해 **오늘의 식단**까지 함께 보여줍니다.

- `index.html` — 화면 (제보 UI + 현황 보드 + 오늘의 식단)
- `server.js` — 로컬 서버 (부산대 식단 파싱 + 제보를 구글시트로 전달 · **설치 패키지·API 키 불필요**)
- `google-apps-script.gs` — 구글시트에 붙여넣는 기록용 스크립트

## 대상 식당 ↔ 부산대 페이지 매핑

| 앱의 식당 | 부산대 식단안내 |
|-----------|-----------------|
| 금정회관(교직원) | 금정회관 교직원 식당 |
| 금정회관(학생) | 금정회관 |
| 학생식당 | 학생회관 |

## 실행 방법

식단 데이터는 외부 사이트(m.pusan.ac.kr)에서 가져오므로, 브라우저 CORS 때문에
**로컬 서버로 실행**해야 합니다. PowerShell에서 이 폴더에 들어가서:

```bash
node server.js
```

그다음 브라우저에서:

```
http://localhost:3000
```

- 왼쪽: 식당 선택 → 혼잡도 1~5 선택 → **제보 등록**
- 오른쪽: 실시간 현황 보드
- 아래: **오늘의 식단** (중식/석식 · 정식/일품 · 가격)

서버를 끄려면 `Ctrl + C`.

> 참고: 혼잡도 제보/현황은 서버 없이 `index.html` 을 직접 열어도 작동하지만,
> **오늘의 식단은 `node server.js` 로 실행해야** 표시됩니다.

## 구글시트 연동 (제보 데이터 수집·분석)

제보를 구글시트에 자동으로 쌓아 **요일·시간대별 혼잡 패턴**을 분석할 수 있습니다.
API 키·서비스 계정 없이, 구글 Apps Script 웹앱만으로 연동합니다.

**1) 시트 + 스크립트 준비**

1. 새 구글시트 생성
2. 상단 **확장 프로그램 → Apps Script**
3. `google-apps-script.gs` 내용을 전부 복사해 붙여넣고 저장

**2) 웹앱으로 배포**

1. Apps Script 화면 우측 상단 **배포 → 새 배포**
2. 유형: **웹 앱**
3. 실행 계정 **나**, 액세스 권한 **모든 사용자**
4. 배포 → 권한 승인 → 나오는 **웹 앱 URL**(`.../exec`) 복사

**3) URL을 `sheet-url.txt` 에 저장하고 실행**

1. 이 폴더에 **`sheet-url.txt`** 파일을 만들고, 복사한 `/exec` URL을 **한 줄**로 붙여넣어 저장
   (이 파일은 `.gitignore` 로 깃허브에 올라가지 않습니다 — URL 노출 방지)
2. **`start.bat` 더블클릭** (또는 `node server.js`)

또는 PowerShell에서 직접 지정:

```bash
$env:SHEET_WEBHOOK_URL = "복사한_/exec_URL"; node server.js
```

이제 제보할 때마다 시트에 `날짜 · 시각 · 요일 · 식당 · 혼잡도(1-5) · 등급` 행이 추가됩니다.
(웹앱 URL을 브라우저에서 직접 열면 정상 동작 여부를 확인할 수 있습니다.)

> `SHEET_WEBHOOK_URL` 을 지정하지 않으면 화면에 "구글시트 미연동 (로컬만 저장)" 으로 표시되고,
> 앱의 나머지 기능은 그대로 작동합니다.

**분석 예시(시트에서):** 요일·시간대별 평균 혼잡도, 피크 시간, 식당별 비교 등을 피벗테이블/차트로.

## 동작 방식

```
[화면] http://localhost:3000
  ├─ 혼잡도 제보  → 현황 보드 갱신 + /api/report → server.js → 구글시트(Apps Script)
  └─ 오늘의 식단  → server.js 가 m.pusan.ac.kr/ko/meals/{geumjeong,pusan} 를
                    가져와 파싱(10분 캐시) → /api/menu 로 제공 → 화면 표시
```

## 옵션 (환경변수)

| 변수 | 설명 | 기본값 |
|------|------|--------|
| `PORT` | 서버 포트 | `3000` |
| `SHEET_WEBHOOK_URL` | 구글 Apps Script 웹앱 `/exec` URL | 없음(미연동) |

## 현재 한계 & 다음 확장

- 다음 단계: 시트/DB에서 **여러 제보를 평균·시간 감쇠로 종합**해 보드에 반영, **요일·시간대 예측**, **카카오맵 지도 표시**, 식단 **날짜 이동**
